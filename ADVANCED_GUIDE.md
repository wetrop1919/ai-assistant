# 🚀 Руководство по Advanced Функциям AI Assistant

## Включение Advanced Режима

```bash
python main.py --advanced --cli

Флаги:

--advanced - включить RAG и мультимодальность
--profile - профилирование производительности
--debug - режим отладки
RAG (Retrieval Augmented Generation)
RAG система позволяет ассистенту работать с вашими собственными документами.

Импорт документов
bash
👤 Вы: /import /path/to/documents
Поддерживаемые форматы:

.txt - текстовые файлы
.pdf - PDF документы
.docx - Word документы
.md - Markdown файлы
Поиск в документах
Ассистент автоматически использует RAG при ответе на вопросы.

Code
👤 Вы: Что сказано в документе о...?
🤖 <ответ с цитатами из документа>
Расширенная Память
Система памяти состоит из трех типов:

1. Episodic Memory (События)
Хранит все события с временными метками.

Code
/memory search <запрос> - поиск по событиям
2. Semantic Memory (Факты)
Хранит знания о пользователе:

Имя
Предпочтения
Проекты
Знания о системе
3. Procedural Memory (Процедуры)
Хранит как что-то делать:

Часто используемые команды
Макросы
Workflow'ы
Мультимодальность
Описание изображений
bash
👤 Вы: /image /path/to/image.jpg
🤖 <подробное описание>
Вопросы по изображениям
bash
👤 Вы: Что на этом изображении?
Извлечение текста (OCR)
bash
👤 Вы: /image /path/to/screenshot.png
🤖 Извлеченный текст: ...
Маршрутизация Запросов
Ассистент автоматически выбирает оптимальный способ обработки:

Уровень	Модель	Примеры
1	Command	/help, /stats
2	Pattern	"привет", "спасибо"
3	Phi-3	Простые вопросы
4	Llama 3	Обычные запросы
5	Llama 3	Сложные рассуждения
6	CodeLlama	Программирование
Профилирование Производительности
bash
python main.py --profile
Выводит метрики:

Время ответа по моделям
Использование памяти
Количество запросов
Примеры Advanced Использования
Работа с кодом + RAG
bash
👤 Вы: На основе наших документов, как реализовать паттерн Factory?
🤖 <объяснение + примеры из ваших документов>
Анализ изображений + память
bash
👤 Вы: /image screenshot.png
👤 Вы: Что на скриншоте?
🤖 Описывает скриншот и запоминает его для контекста
Автоматизация с памятью
bash
👤 Вы: Запиши макрос для часто используемого процесса
👤 Вы: Повтори этот макрос
🤖 Выполняет сохраненные действия
Статус Ассистента
bash
👤 Вы: /status
🤖 Показывает:
   - Здоровье компонентов
   - Время работы
   - Загруженные модели
   - Статистику памяти
Продвинутые Команды
bash
/memory search <текст>    - умный поиск в памяти
/import <папка>           - импорт документов для RAG
/image <путь>             - анализ изображения
/status                   - статус ассистента
/skills                   - список и статус навыков
/export                   - экспорт всех данных
Производительность
Оптимизация
Используйте --cli для скорости (без голоса)
Кэширование: Первый запрос медленнее, последующие быстрее
RAG: Помогает дать точные ответы, но требует больше памяти
Мониторинг
Метрики доступны в Prometheus формате:

Code
http://localhost:8000/metrics
Поиск и Устранение Неисправностей
RAG не работает
bash
- Проверьте что ChromaDB установлена: pip install chromadb
- Импортируйте документы: /import /path
Медленные ответы
bash
- Используйте --profile для анализа
- Проверьте статус Ollama: /status
- Уменьшите размер контекста в config
Ошибки памяти
bash
- Проверьте место на диске
- Экспортируйте и очистите: /export затем /memory clear

## 🎯 ФИНАЛЬНЫЕ ИЗМЕНЕНИЯ

Добавьте в `config.py`:

```python
# Advanced режим
ENABLE_RAG = Field(default=False, env="ENABLE_RAG")
ENABLE_MULTIMODAL = Field(default=False, env="ENABLE_MULTIMODAL")
ENABLE_PROMETHEUS = Field(default=False, env="ENABLE_PROMETHEUS")
И в .env.example:

bash
ENABLE_RAG=false
ENABLE_MULTIMODAL=false
ENABLE_PROMETHEUS=false
